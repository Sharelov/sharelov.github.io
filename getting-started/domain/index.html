Terminology and Domain Logic

Our domain is divided in six main components:

  • Brand
  • Campaign
  • Publication
  • Subscription
  • User
  • Membership (Teammates)

And some secondary ones:

  • Social Accounts
  • Permissions
  • Authorizations
  • Tags
  • Files
  • Services (external API)

Is important to follow this terminology throughout the the project in order to maintain consistency. If new terminology is needed, it should be discussed collectively first.

Brand

Model: Brand

A Brand is a group that contains multiple Campaigns and it belogns to a Subscription. There are also other components that can be associated to a Brand:

  • Teammates (Memberships): The authorized users can authorize other teammates to work on a Brand.
  • Social Accounts: Social Media Accounts need to be assigned to the Brand so they can be available in that Brand's Campaigns.
  • Tags: These are basically categories used to group Publications. Brand tags are available to all Publications in that Brand.
  • Files: images.

Campaigns

Model: Campaign

A Campaign is a collection of Publications (posts, tweets...) and it belongs to a Brand. There are other components that can be associated to a Campaign:

  • Social Accounts: Depending of the type, a Campaign can have one (1) Social Media Account associated to it.
  • Client Memberships: Clients cannot see a Campaigns unless they are included as a teammate (or authorized [📄]) to the it.
  • Tags: They are the same as Brand tags, but only available to the Publications of the Campaign they are associated to.

Publications

Base Model: Publication

Specific Models: Publications/*

A Publication belongs to a Campaign and is what is eventually sent to a Social Account (or published).

A publication may have multiple components associated to it:

  • Files: images, videos, etc.
  • Placeholders: A Placeholder is a special type o image created dynamically with the text that user specifies.
  • Tag: A Publication can belong to only one, either Brand or Campaign, Tag group.
  • Specific Components: Depending of the Publication, some other components can be associated also.

User

Model: User

An User is the container for the user information (email, name, last name...). Since Sharelov is a subscription based app, the user could potentially belong to different Subscriptions, so, in order not to repeat the user information, we use the pivot Membership.

Subscription

Model: Subscription

A Subscription it's the primary container where all other components live.

A Subscription is basically a Sharelov account. The only way to create a Subscription is to fill up Sharelov's sign up form.

There can be only one owner Membership and one User per Subscription.

Membership

Model: Membership

A Membership is the association (pivot) between the Subscription and the User.

An User can have multiple Memberships, and each Memberships belongs to only one Subscription.

A Brand Team consists of a collection of Memberships assigned to that Brand.
